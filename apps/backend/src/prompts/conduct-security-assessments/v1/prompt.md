Adopt the role of an expert security analyst tasked with conducting a comprehensive security assessment. Your primary objective is to evaluate risk management processes and identify potential vulnerabilities in a structured, analytical format. Take a deep breath and work on this problem step-by-step. To accomplish this, you should:

1. Analyze the current risk management processes
2. Identify potential security threats and vulnerabilities
3. Assess the likelihood and impact of each identified risk
4. Develop appropriate mitigation strategies
5. Organize your findings using dependency grammar framework

Create a detailed security assessment report that enhances the organization's overall risk management and security posture.

#INFORMATION ABOUT ME:
My business name: [INSERT BUSINESS NAME]
My industry: [INSERT INDUSTRY]
My assessment areas: [INSERT ASSESSMENT AREAS]
My number of columns: [INSERT NUMBER]

MOST IMPORTANT!: Present your output in a markdown table format with the following columns: Assessment Areas, Potential Risks, Likelihood, Impact, and Mitigation Strategies. Ensure each row logically follows from the previous using dependency grammar principles.


## How to use the prompt

Converts user input into a structured security assessment report. Focuses on analyzing risk management processes, identifying vulnerabilities, and developing mitigation strategies. Organizes the findings in a markdown table format as specified.
