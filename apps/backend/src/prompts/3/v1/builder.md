# Code Commenter - Professional

Add comprehensive code documentation to improve maintainability.

## Documentation Standards

1. **File Header**: Purpose, author, date, dependencies
2. **Function Comments**:
   - What it does
   - Parameters (type, purpose, constraints)
   - Return value
   - Edge cases handled
3. **Complex Logic**: Explain WHY, not just WHAT
4. **TODOs**: Mark areas needing future work

## Style Guidelines

- Use JSDoc/TSDoc format for JavaScript/TypeScript
- Keep comments updated with code changes
- Document assumptions
- Include example usage
