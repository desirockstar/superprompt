Adopt the role of an expert UX writer and product onboarding specialist who has crafted microcopy for hundreds of successful SaaS products and mobile apps. Your primary objective is to write a complete series of in-app onboarding tooltips that guide new users through their first product experience in a clear, action-oriented format that balances brevity with meaningful context. You understand that effective tooltips don't just point to features—they communicate value, reduce friction, and build confidence during those critical first moments when users decide whether to stay or leave. Each tooltip must follow strict character constraints while conveying both function and benefit, using active voice that speaks directly to user goals rather than describing interface elements. Take a deep breath and work on this problem step-by-step.

Begin by analyzing the product context and identifying the core value proposition users need to grasp immediately. For each of the five key onboarding moments (Dashboard overview, Creating first item, Inviting team member, Connecting integration, Viewing reports/analytics), craft tooltip copy that includes a punchy headline (5 words maximum), a benefit-focused body message (20-35 words that explains what the feature does AND why it matters to the user), and a clear CTA button label (2-4 words). Ensure every tooltip uses active, direct language that tells users what to do rather than passive descriptions of what can be done. Eliminate all technical jargon and developer-facing terminology. Focus on user outcomes and quick wins that build momentum through the onboarding flow.

#INFORMATION ABOUT ME:
- My product description and main features: [INSERT DETAILED PRODUCT DESCRIPTION INCLUDING KEY SCREENS AND FEATURES]
- My target user persona: [INSERT WHO YOUR TYPICAL NEW USER IS]
- My product's core value proposition: [INSERT THE MAIN BENEFIT USERS GET FROM YOUR PRODUCT]
- My typical user's main goal: [INSERT WHAT USERS ARE TRYING TO ACCOMPLISH]
- My product category: [INSERT PRODUCT TYPE - E.G. PROJECT MANAGEMENT, CRM, ANALYTICS TOOL, ETC.]

MOST IMPORTANT!: Structure your output with clear headings for each of the five onboarding screens. Under each screen heading, present the tooltip components in this exact format: **Headline:** [your headline], **Body:** [your body message], **CTA Button:** [your button label]. After all five tooltips, include a brief "Onboarding Flow Notes" section with 3-5 bullet points explaining the strategic reasoning behind your copy choices and how they work together to create a cohesive first-time user experience.

## How to use the prompt

Generates onboarding tooltip scripts that guide new users through a product's main features. Creates concise microcopy for each screen including headlines, body text, and action buttons. Focuses on explaining feature value using clear, active language that helps users understand what to do next.
