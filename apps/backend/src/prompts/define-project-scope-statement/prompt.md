Adopt the role of an expert project manager tasked with defining a project scope statement. Your primary objective is to create a comprehensive and structured project scope statement using dependency grammar framework. Take a deep breath and work on this problem step-by-step. Begin by clearly stating the project objectives, then outline the deliverables, assumptions, constraints, and exclusions. Ensure that each element is logically connected and supports the overall project goals. Use precise language to avoid ambiguity and potential scope creep. Consider potential risks and challenges specific to the industry and incorporate them into your statement.

#INFORMATION ABOUT ME:
My project: [INSERT PROJECT NAME]
My industry: [INSERT INDUSTRY]
My key stakeholders: [LIST KEY STAKEHOLDERS]
My project timeline: [INSERT PROJECT TIMELINE]
My budget constraints: [INSERT BUDGET CONSTRAINTS]

MOST IMPORTANT!: Provide your output in a structured format with clear headings for each section (Objectives, Deliverables, Assumptions, Constraints, and Exclusions) and use bullet points for individual items within each section.


## How to use the prompt

Converts user input into a structured project scope statement using dependency grammar. Organizes the statement into clear sections: Objectives, Deliverables, Assumptions, Constraints, and Exclusions. Ensures precision and clarity in language to prevent scope creep and address industry-specific risks.
