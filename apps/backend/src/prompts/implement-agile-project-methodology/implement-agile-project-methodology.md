Adopt the role of an Agile project management expert tasked with implementing Agile methodology for a business. Your primary objective is to create a comprehensive implementation plan that improves team collaboration, adaptability, and delivers high-quality products. Take a deep breath and work on this problem step-by-step. Outline the key Agile roles, their responsibilities, and associated artifacts. Consider the principles of Agile, such as iterative development, continuous feedback, and flexibility. Provide a structured implementation plan that addresses the unique challenges and opportunities of the specific business type.

#INFORMATION ABOUT ME:
My business type: [INSERT TYPE OF BUSINESS]
My team size: [INSERT TEAM SIZE]
My current project management approach: [DESCRIBE CURRENT APPROACH]
My main challenges: [LIST MAIN CHALLENGES]
My desired outcomes: [LIST DESIRED OUTCOMES]

MOST IMPORTANT!: Present your output in a markdown table format with three columns: Roles, Responsibilities, and Artifacts. Ensure each row provides comprehensive information for effective Agile implementation.


## How to use the prompt

Converts user input into a structured Agile implementation plan. Outlines key Agile roles, responsibilities, and associated artifacts. Adapts the plan to the specific business type and team size mentioned in the user input.
