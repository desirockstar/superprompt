Adopt the role of a project management expert tasked with creating a Work Breakdown Structure (WBS). Your primary objective is to develop a comprehensive and well-structured WBS for a specific project in a given industry. Take a deep breath and work on this problem step-by-step. Begin by understanding the project scope and objectives. Then, break down the project into main deliverables, sub-deliverables, and individual tasks. Ensure each task is specific, measurable, and has a clear outcome. Consider the logical sequence of tasks, dependencies, and resource requirements. Assign unique Task IDs to each item in the WBS for easy reference and tracking.

#INFORMATION ABOUT ME:
My project: [INSERT PROJECT NAME]
My industry: [INSERT INDUSTRY]
My project timeline: [INSERT PROJECT TIMELINE]
My key deliverables: [INSERT KEY DELIVERABLES]
My available resources: [INSERT AVAILABLE RESOURCES]

MOST IMPORTANT!: Present your output in a markdown table format with columns for Task ID, Task Name, Duration, Start Date, and End Date. Ensure the table is well-organized, easy to read, and provides a clear overview of the project structure.


## How to use the prompt

Converts user input into a structured Work Breakdown Structure (WBS) for project management. Breaks down the project into main deliverables, sub-deliverables, and individual tasks, ensuring each is specific and measurable. Organizes tasks into a markdown table format with columns for Task ID, Task Name, Duration, Start Date, and End Date.
