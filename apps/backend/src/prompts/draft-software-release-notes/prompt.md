Adopt the role of an expert software release manager tasked with generating comprehensive software release notes. Your primary objective is to create clear, informative, and well-structured release notes that effectively communicate changes, improvements, and fixes to stakeholders in a concise and organized format. To achieve this, you should:

1. Gather information about the new features, enhancements, bug fixes, and known issues.
2. Categorize the changes into logical sections (e.g., New Features, Improvements, Bug Fixes, Known Issues).
3. Provide concise yet descriptive explanations for each item.
4. Include version numbers, release dates, and any relevant system requirements.
5. Use consistent formatting and language throughout the document.
6. Prioritize the most important changes and place them at the beginning of each section.
7. Include any necessary instructions or steps for users to take advantage of new features or address known issues.

Take a deep breath and work on this problem step-by-step to ensure comprehensive and accurate release notes.

#INFORMATION ABOUT ME:
My software name: [INSERT SOFTWARE NAME]
My version number: [INSERT VERSION NUMBER]
My release date: [INSERT RELEASE DATE]
My target audience: [INSERT TARGET AUDIENCE]
My key changes: [LIST KEY CHANGES, FEATURES, OR FIXES]

MOST IMPORTANT!: Provide your output in a structured format with clear headings and bullet points for each section to ensure maximum readability.

## How to use the prompt

Gathers detailed information about new features, enhancements, bug fixes, and known issues for software release notes. Organizes changes into logical sections with clear headings for easy understanding. Provides concise and descriptive explanations, including version numbers, release dates, and system requirements.
