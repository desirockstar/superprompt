🧭 Recommended Order
1. Business Analysis Plan

Start here—this defines how you’ll do everything else.

Scope, stakeholders, techniques, timelines
👉 Without this, everything downstream is ad hoc
2. SWOT Analysis

Do early strategic grounding.

Understand market, risks, positioning
👉 Prevents building the wrong thing
3. Requirements Gathering

Now collect actual needs.

Stakeholders, users, constraints
👉 Informed by SWOT → more focused inputs
4. Flow Charts

Translate requirements into system understanding.

Process flows, user journeys
👉 Helps remove ambiguity before breaking into stories
5. User Story Generator

Now convert structured understanding into Agile units.

Epics → User Stories
👉 Doing this earlier = garbage stories
6. Acceptance Criteria

Define “done” for each story.

Functional + edge conditions
👉 Must come after stories exist
7. Product Roadmap Maker

Now decide sequencing over time.

MVP → phased releases
👉 Based on validated stories + priorities
8. JQL Generation (for Jira)

This is purely execution/query optimization, not product definition.

Used to filter, track, and manage issues
👉 Comes after backlog exists
⚡ Final Flow (Condensed)

Business Plan → SWOT → Requirements → Flows → User Stories → Acceptance Criteria → Roadmap → JQL

⚠️ Quick Pushback
“User Story Generator” before flows/requirements = low-quality backlog
“Roadmap” before stories = guesswork strategy
“JQL” is not a product activity—it’s operational tooling
